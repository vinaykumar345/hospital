# Module 30: Final Production Hardening

## Understand

Final production hardening adds security policy, operational hardening checklist, backup/restore guidance, privacy guidance, and release-readiness verification.

## Security Rules

- Secrets must never be committed.
- TLS, encryption at rest, RBAC, audit logging, tenant isolation, input validation, and monitoring must be enabled in production.
- AI outputs remain assistive and review-required.

Run `npm run test` from the repository root.
